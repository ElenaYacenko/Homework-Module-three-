## Технологический стек

| Инструмент | Назначение                                      |
|------------|-------------------------------------------------|
| `Java 21`  | Язык программирования                           |
| `Selenide` | UI-фреймворк (обёртка над Selenium WebDriver)   |
| `JUnit 5`  | Тест-раннер и система утверждений               |
| `Gradle`   | Сборка проекта и управление зависимостями       |
| `Chrome`   | Браузер для выполнения тестов                   |

##             Список автоматизированных тестов

| Метод теста | Тип | Краткое описание |
|-------------|-----|-----------------|
| `successfulFullFormFillTest` | ✅ Позитивный | Полное заполнение всех полей формы, проверка данных в модальном окне |
| `requiredFieldsOnlyFormFillTest` | ✅ Позитивный | Заполнение только обязательных полей, успешная отправка |
| `negativeTest_EmptyRequiredFieldsShowRedBorders` |  Негативный | Отправка пустой формы, проверка красных рамок у обязательных полей |
| `negativeTest_InvalidPhoneNumberShowsRedBorder` | ❌ Негативный | Ввод 9-значного номера в поле телефона, проверка валидации |
| `negativeTest_PhoneNumberWithLettersShowsRedBorder` | ❌ Негативный | Ввод букв в поле телефона, проверка подсветки ошибки |
| `negativeTest_MissingFirstNameShowsRedBorder` | ❌ Негативный | Отправка формы без заполнения First Name (пустое поле) |
| `negativeTest_MissingLastNameShowsRedBorder` | ❌ Негативный | Отправка формы без заполнения Last Name (пустое поле) |
| `negativeTest_MissingMobileShowsRedBorder` | ❌ Негативный | Отправка формы без заполнения поля телефона |
| `negativeTest_UnselectedGenderShowsValidationError` | ❌ Негативный | Отправка формы без выбора пола |
| `negativeTest_LastNameExceedsMaxLengthShowsRedBorder` | ❌ Негативный | Ввод 260 символов в поле фамилии, проверка отклонения |
| `negativeTest_FirstNameOneCharacterRejected` | ❌ Негативный | Ввод 1 символа в First Name (проверка минимальной длины) |

##             Детализация тест-кейсов

<details>
<summary><strong>✅ TC-AUTO-001: Успешное заполнение всех полей формы</strong></summary>

**ID:** TC-AUTO-001  
**Название тест-кейса:** Успешная регистрация с полным заполнением формы  
**Предусловия:**
- Браузер открыт, страница `/automation-practice-form` доступна
- Пользователь не авторизован

**Шаги:**
1. Открыть форму регистрации
2. Заполнить `First Name`: "Elena"
3. Заполнить `Last Name`: "Ya"
4. Заполнить `Email`: "xxx@ya.ru"
5. Выбрать `Gender`: Female
6. Заполнить `Mobile`: "9999999999"
7. Выбрать `Date of Birth`: 30 March 2020 (через календарь)
8. Добавить `Subjects`: Maths
9. Выбрать `Hobbies`: Reading
10. Заполнить `Current Address`: "RU"
11. Выбрать `State`: Haryana, `City`: Karnal
12. Нажать кнопку **Submit**

**Ожидаемый результат:**
- Открыто модальное окно с заголовком "Thanks for submitting the form"
- В таблице модального окна отображаются все введённые данные:
    - Student Name: Elena Ya
    - Student Email: xxx@ya.ru
    - Gender: Female
    - Mobile: 9999999999
    - Date of Birth: 30 Mar,2020
    - Subjects: Maths
    - Hobbies: Reading
    - Address: RU
    - State and City: Haryana Karnal

**Примечания:**
- Используется селектор `$x("//tr[td[contains(., 'Label')]]/td[2]")` для проверки таблицы
- Для выбора даты применяется календарь через `.react-datepicker__*` селекторы

</details>

**Шаги:**

- Для Gender проверяется отсутствие атрибута `checked`
