# Параметризованные тесты

##  Описание задания

Автоматизация тестирования формы входа в личный кабинет с использованием параметризованных тестов JUnit 5.

**Цель:** Написать веб-тесты с использованием минимум 3 различных датапровайдеров для проверки валидации полей формы входа.

---

## Технологии и инструменты

- **Java 21**
- **JUnit 5** (Jupiter) — фреймворк для тестирования
- **Selenide 7.2.0** — обёртка над Selenium WebDriver
- **WebDriverManager 5.6.0** — управление драйверами браузеров
- **Gradle** — система сборки
- **Chrome** — браузер для запуска тестов

---

## Тестовые сценарии

### 1. Валидация поля "Телефон или электронная почта" (`@CsvSource`)

**Количество тестовых наборов:** 13

Проверка различных невалидных форматов ввода с разными ожидаемыми ошибками:

| Тип данных | Пример | Ожидаемая ошибка |
|------------|--------|------------------|
| Пустая строка | `''` | Обязательное поле! |
| Пробелы | `'   '` | Введите телефон или почту |
| Короткий телефон | `7900900909` | Введенный номер телефона не соответствует допустимому формату |
| Длинный телефон | `790090090909` | Введенный номер телефона не соответствует допустимому формату |
| Только буквы | `phonenumber` | Введите телефон или почту |
| Кириллица | `номертелефона` | Введите телефон или почту |
| Спецсимволы | `123654@#$` | Введите телефон или почту |
| Несуществующий пользователь | `8(999)666-32-32` | Пользователь не найден |
| Невалидный email | `xxx123ya.ru` | Введите телефон или почту |
| Email без @ | `abcdef` | Введите телефон или почту |
| Email без домена | `@domain.рф` | Введите телефон или почту |
| Email без имени | `user@.рф` | Введите телефон или почту |
| Email без зоны | `user@domain` | Введите телефон или почту |

**Проверки:**
- ✅ Текст ошибки соответствует ожидаемому
- ✅ Цвет текста ошибки: `rgba(227, 0, 0, 1)` (красный)

### 2. Валидация поля "Пароль" (`@ValueSource`)

**Количество тестовых наборов:** 6

Проверка различных невалидных форматов пароля:

| Тестовые данные | Описание |
|-----------------|----------|
| `""` | Пустая строка |
| `"      "` | Только пробелы |
| `"123456"` | Только цифры |
| `"abcdef"` | Только латинские буквы |
| `"пароль"` | Только кириллица |
| `"@#$%"` | Только спецсимволы |

**Проверки:**
- ✅ Цвет текста ошибки: `rgba(227, 0, 0, 1)` (красный)

---

### 3. Проверка связки логин-пароль (`@CsvFileSource`)

**Количество тестовых наборов:** 5 (из файла `login-validation-extended.csv`)

Тестовые данные хранятся в отдельном CSV-файле:

| Логин | Пароль | Ожидаемая ошибка |
|-------|--------|------------------|
| *(пусто)* | *(пробел)* | Обязательное поле! |
| *(пусто)* | @#456ASDgh | Обязательное поле! |
| 79000000000 | 123456ASDgh | Пользователь не найден |
| 79999999999 | @#456ASDgh | Неправильный пароль |
| 79999999999 | *(пробел)* | Неправильный пароль |

**Особенности:**
- Используется поиск элемента по тексту ошибки (`byText`)
- Проверяется видимость элемента и цвет текста
